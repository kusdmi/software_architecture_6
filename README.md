# Оптимизация скрипта подсчёта частоты слов

## Исходная задача

Скрипт загружает веб-страницу, считывает список слов из файла `words.txt` и для каждого слова подсчитывает количество его вхождений в текст страницы. Результат выводится в виде словаря.

## Выявленные проблемы производительности

1. **Многократные сетевые запросы**  
   Функция подсчёта частоты для каждого слова загружает страницу заново. Если в `words.txt` 10 слов, страница скачивается 10 раз.

2. **Повторное разбиение текста**  
   Для каждого слова текст разбивается на список слов (`split()`), хотя результат одинаков для всех слов.

3. **Неэффективный подсчёт**  
   Каждое слово ищется линейным проходом по всем словам страницы. Сложность `O(N * M)`, где `N` – количество слов на странице, `M` – количество искомых слов.

4. **Отсутствие обработки ошибок и таймаутов**  
   При проблемах с сетью скрипт может зависнуть или завершиться с непонятной ошибкой.

## Профилирование

С помощью `cProfile` установлено, что значительная часть времени (~0.37 с для трёх слов) тратится на SSL-рукопожатия и многократные сетевые вызовы. Три вызова `count_word_frequencies` занимают 0.209 с, причём основное время уходит на сетевые операции.

## Проведённые оптимизации

1. **Однократная загрузка страницы**  
   Страница загружается один раз, а текст сохраняется в переменную. Все дальнейшие операции работают с локальной копией.

2. **Использование `collections.Counter`**  
   `Counter` позволяет за один проход по списку слов подсчитать частоты всех слов. Сложность снижается до `O(N)` для подсчёта и `O(M)` для выборки нужных слов.

3. **Очистка слов от знаков препинания и приведение к нижнему регистру**  
   Повышает точность подсчёта (например, «aging» и «aging,» теперь считаются одинаково). Реализовано через функцию `clean_word()`.

4. **Добавление таймаута и обработки ошибок**  
   Установлен таймаут 10 секунд при запросе, проверяется статус ответа, обрабатываются возможные исключения (сетевая ошибка, недоступность сервера, отсутствие файла).

5. **Устранение повторных вызовов `split()`**  
   Текст разбивается на слова один раз, результат сохраняется в списке.
